URLopedia Lives!

Note: this is a very rough initial draft of the URLopedia user guide and will likely evolve considerably over the next few months. To that end, please edit/correct/augment liberally, especially in those areas labeled "TK."

A Uniform Resource Locator, more commonly known as a URL or web address, is how we find sites online. For instance, type "http://facebook.com" into a web browser. If everything works as intended, you’ll soon be checking out your friends’ funny cat photos. But what if instead of visiting Facebook, you want to learn about the site? For instance, where is it censored? Is it archived in the Internet Archive’s Wayback Machine?

It turns out that there are a lot of resources for learning about URLs. But it’s hard to keep track of them all! That’s where URLopedia comes in. It is a very lightweight service for combining the collective knowledge we have about any given URL. Using publicly available APIs from a number of different services, URLopedia shows you the data we have about certain websites. All you need to do is enter a URL and URLopedia will do the rest, providing you with JSON objects that contain all of the information that we could find.

Over the coming months URLopedia hopes to grow and add new APIs, and to make it possible to search URLs directly from your browser. On this site you can see:

What data is in URLopedia?

Currently, URLopedia uses APIs available from the following organizations:

  • Describ.ing.am: TK
  • Herdict: real-time censorship data
  • Media Cloud: site content analysis and mapping
  • The Wayback Machine: the Internet Archive database of cached sites

How to install URLopedia

URLopedia is fairly easy to install, even for someone with little familiarity with JS. First things first: install node.js. You can download either the source code or an installer from the node.js website or install from the command line (OSX: brew install node, Linux: sudo apt-get install nodejs npm).

You’ll need a GitHub account for the next step, which is to clone (or fork & clone) https://github.com/berkmancenter/urlopedia. New to GitHub? No problem:

  1. Sign up for an account at GitHub.
  2. Install and set up Git on your local machine (here's a handy tutorial).
  3. Clone the URLopedia repository onto your computer with this command:
    git clone https://github.com/berkmancenter/urlopedia.git

The next step is to change to the cloned directory and get the dependencies we need to run URLopedia. Once you’re inside your new local urlopedia directory, run this command:

npm install

That’s it—you’re all set up! To run URLopedia, run this command:

node server.js

(Using Linux? Use nodejs server.js instead)

Let’s test it to make sure it’s working. Open a web browser and go to http://localhost:3000/all?http://facebook.com. You should see something like this:

TK (JSON results)

All done? Type Ctrl-C to exit.

How to query URLopedia

The most basic search query for URLopedia starts like this:

http://localhost:3000/all?

A query starting with this string will return data from the following sources:

  • [TK]
  • [TK are not included because TK]

Let’s say you want all of the basic information URLopedia has about Facebook. You would add Facebook’s URL—http://facebook.com—to this basic starting point to get the following query:

http://localhost:3000/all?http://facebook.com

Maybe you only care about Herdict data for Facebook. To filter your results down to only the information URLopedia has from Herdict, you would change your query to read:

http://localhost:3000/herdict?http://facebook.com

A quick cheat sheet of data sources and appropriate queries:

Describing.am
http://localhost:3000/describing?
Herdict
http://localhost:3000/herdict?
Media Cloud
http://localhost:3000/mediacloud?
Wayback Machine
http://localhost:3000/wayback?
